class mumodo.corpus.Resource(**kwargs)[source]¶

A mumodo resource abstract class

Resources can be any form of data (e.g. video, audio, motion capture)

The object describes the resource, how it can be retrieved from a file, and a sensible way to display the data, or cut a slice of the data. This is is a base class used to derive children for specific resource types.

However, this class is useful for naming resources that are not covered by the children classes (yet) but should neverthelees be part of a mumodo.

class mumodo.corpus.CSVResource(**kwargs)[source]¶

A CSV resource coming from a CSV file

The CSV resource represents data stored in CSV files, that are not StreamFrames or Tiers. For those you can use CSVStreamResource and CSVTierResource, respectively.

Mumodo uses the Pandas library to handle CSV files, and will load any CSV file into a Pandas DataFrame. Check the documentation of pandas for more details.

In addition to the base Resource class constructor, this constructor accepts the following additional kwargs:

kwargs - a dictionary of kwargs that are passed to the Pandas
DataFrame.from_csv constructor

The CSV resource cannot be sliced by time

class mumodo.corpus.BaseTierResource(**kwargs)[source]¶

Base Tier Resource

The BaseTierResource represents Interval or Point Tier resources, and serves as a parent for children that load tiers from different file types. IntervalTiers and PointTiers are Pandas DataFrames with specific column names and structure. See the documentation of mumodoIO for more information.

class mumodo.corpus.AudioResource(**kwargs)[source]¶

A mumodo Audio resource object

An AudioResource is a type of resource based on Audio files. This module uses moviepy.editor to handle audio files and is thus limited to the formats supported by that package. This is a rich set of formats, as moviepy is based on ffmpeg.

__init__(**kwargs)[source]¶

AudioResource Constructor

player – an external player (e.g. VLC, mplayer) that can play
the audio files. The value must be the FULL path to the executable, or the executable has to be in the PATH
channel – In case of multi-channel audios, an identifier
(usually a number) of the channel

class mumodo.corpus.XIOStreamResource(**kwargs)[source]¶

StreamFrame from XIO file

This class implements a StreamFrame resource that is loaded from an XIO file.

In addition to the base Resource kwargs, the constructor accepts additional kwargs:

argument in order to open a streamframe from an XIO file
kwargs – a dictonary of kwargs to be passed to the function
that parses the XIO file and returns the StreamFrame. Notably, an offset can be one of these. See the documentation of mumodo.mumodoIO for more information

class mumodo.corpus.Mumodo(**kwargs)[source]¶

The mumodo corpus object

__getitem__(item)[source]¶

Access a resource by its name

For convenience, the following expression will store a reference of the resource named ‘myresource’

>>> r = mymodo['myresource']

Numerical access to resources is not supported, as it is not really needed.
